3.1 Quais so as variveis? Quais so os tipos de variveis (discreta, categrica, contnua)?

Variveis do Dataset

Varivel Tipo Descrio
type Categrica Tipo do vinho ("white" para vinho branco ou "red" para vinho tinto).
fixed acidity Contnua Acidez fixa do vinho, medida em g/dm. Representa os cidos no volteis, como o cido tartrico.
volatile acidity Contnua Acidez voltil, medida em g/dm. Refere-se aos cidos que evaporam facilmente, como o cido actico. Altos valores podem indicar sabor avinagrado.
citric acid Contnua Quantidade de cido ctrico no vinho, medida em g/dm. Ajuda a dar frescor ao vinho.
residual sugar Contnua Acar residual do vinho, medido em g/dm. Refere-se ao acar que sobra aps a fermentao. Vinhos secos geralmente tm menos de 4 g/dm.
chlorides Contnua Teor de cloretos (sal), medido em g/dm. Influencia o sabor salgado do vinho.
free sulfur dioxide Contnua Dixido de enxofre livre, medido em mg/dm. Ajuda a prevenir a oxidao e crescimento de microrganismos.
total sulfur dioxide Contnua Dixido de enxofre total, medido em mg/dm. Inclui o SO livre e o SO ligado. Regulado por normas para evitar excesso.
density Contnua Densidade do vinho, em g/cm. Relacionada ao teor alcolico e teor de acar.
pH Contnua Medida da acidez geral do vinho. Um pH baixo indica maior acidez, enquanto um pH alto indica menor acidez. A maioria dos vinhos varia entre 3 e 4.
sulphates Contnua Sulfatos, medidos em g/dm. Influenciam a estabilidade microbiolgica do vinho.
alcohol Contnua Teor alcolico do vinho, expresso em % (porcentagem de lcool por volume).
opinion Categrica Nova varivel criada a partir da quality. Assume: 0 se qualidade 5 e 1 se qualidade > 5.

4.a Descreva as estapas necessrias para criar um modelo de classificao eficiente.

Etapas para Criar um Modelo de Classificao

Etapa Definio
1. Coleta e Entendimento dos Dados Explorar os dados para entender suas caractersticas, variveis, tipos e distribuies. Identificar possveis problemas, como dados ausentes e outliers.
2. Pr-processamento dos Dados Limpeza dos dados, tratamento de valores ausentes, remoo ou substituio de outliers e normalizao/ padronizao das variveis.
3. Criao da Varivel-Alvo No nosso caso, a varivel "opinion" foi criada a partir de "quality", transformando o problema em classificao binria.
4. Anlise Exploratria dos Dados (EDA) Criar histogramas, grficos de disperso, boxplots e matrizes de correlao para entender relaes entre as variveis e a varivel-alvo.
5. Diviso dos Dados em Treino e Teste Separar os dados em conjunto de treino (80%) e conjunto de teste (20%) para avaliar o desempenho do modelo.
6. Escolha do Algoritmo Escolher um modelo adequado para classificao, como Regresso Logstica, Random Forest, SVM, KNN ou Redes Neurais. Testar diferentes algoritmos para encontrar o melhor.
7. Treinamento do Modelo Ajustar o modelo aos dados de treino e testar diferentes hiperparmetros para otimizao.
8. Avaliao do Modelo Avaliar a performance do modelo nos dados de teste usando mtricas como Acurcia, Preciso, Recall, F1-Score e Matriz de Confuso.
9. Otimizao do Modelo Ajustar hiperparmetros, balancear classes (caso necessrio) e aplicar tcnicas como feature selection para melhorar a preciso do modelo.
10. Implementao e Monitoramento Se o modelo atingir um bom desempenho, ele pode ser implementado para prever novas amostras. O monitoramento contnuo necessrio para garantir sua eficcia ao longo do tempo.

# Definindo funo para calcular mtricas de validao cruzada
def avaliar_modelo(modelo, X, y):
    scores = {
        'accuracy': cross_val_score(modelo, X, y, cv=kfold, scoring='accuracy', n_jobs=-1),
        'precision': cross_val_score(modelo, X, y, cv=kfold, scoring='precision', n_jobs=-1),
        'recall': cross_val_score(modelo, X, y, cv=kfold, scoring='recall', n_jobs=-1),
        'f1_score': cross_val_score(modelo, X, y, cv=kfold, scoring='f1', n_jobs=-1)
    }
    
    resultados = {
        'Mdia Acurcia': np.mean(scores['accuracy']),
        'Desvio Acurcia': np.std(scores['accuracy']),
        'Mdia Preciso': np.mean(scores['precision']),
        'Desvio Preciso': np.std(scores['precision']),
        'Mdia Recall': np.mean(scores['recall']),
        'Desvio Recall': np.std(scores['recall']),
        'Mdia F1-Score': np.mean(scores['f1_score']),
        'Desvio F1-Score': np.std(scores['f1_score'])
    }
    
    return pd.DataFrame([resultados])

Justificativa da Questo 5.

A curva ROC mede o desempenho do modelo em termos da taxa de verdadeiros positivos (TPR) contra a taxa de falsos positivos (FPR) para diferentes limiares de deciso. O AUC (rea Sob a Curva) quantifica a capacidade do modelo de distinguir entre classes.

Escolha do Melhor Modelo

O modelo com a maior rea sob a curva (AUC) o que apresenta melhor desempenho para a tarefa de classificao.

  • Se um modelo tem um AUC maior, significa que ele melhor em separar as classes.
  • Se as curvas de dois modelos forem prximas, a escolha pode depender de outros fatores como interpretabilidade e tempo de treinamento.

Concluso

  • Se a Regresso Logstica ou SVM tiverem AUC maior, um deles deve ser escolhido, pois so mais estveis para classificao.
  • Se a rvore de Deciso apresentar desempenho similar, pode ser escolhida se houver necessidade de interpretabilidade.

6. Resposta da questo 6.

Ele funciona da mesma forma para essa nova base?

No.

Justifique.

O modelo pode no generalizar bem, porque foi treinado apenas com vinhos brancos e pode no capturar corretamente as caractersticas dos vinhos tintos.

Motivos para no usar o mesmo modelo diretamente

  1. Diferenas nas caractersticas dos vinhos:

    • Vinhos tintos e brancos tm composies qumicas diferentes (pH, acidez, acar, lcool, etc.).
    • O modelo treinado s com vinhos brancos pode ter aprendido padres especficos desses vinhos, que no se aplicam aos tintos.

  2. Risco de vis e baixa acurcia:

    • Como os dados de vinho tinto no foram usados no treinamento, o modelo pode errar mais na previso dos vinhos tintos.
    • Se a distribuio dos atributos for muito diferente, o modelo pode estar "fora da sua zona de conforto".

  3. O modelo pode no generalizar bem:

    • Modelos de machine learning funcionam melhor quando o treinamento reflete bem os dados de inferncia.
    • O ideal seria treinar um modelo com dados de ambos os vinhos para melhorar a generalizao.

O que fazer ento? Treinar um novo modelo combinando vinhos brancos e tintos no treinamento. Desta forma, o modelo aprende caractersticas de ambos os tipos de vinho. A previso ser mais precisa e confivel para os vinhos tintos.
